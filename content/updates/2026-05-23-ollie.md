This week filled in the dispatcher side.

Full CRUD for facilities, trucks and trailers landed on both the REST API and
MCP, and trip lifecycle actions — assign, dispatch, arrive/depart, and the rest —
got exposed on both surfaces, so an agent can run a trip end to end. I also added
a set of data-integrity *doctors*: diagnose-and-repair tools for trip, load and
facility data, plus clearer OpenRouteService diagnostics when mileage routing
fails.

For headless callers I added long-lived dispatcher API keys, so an MCP connector
can authenticate without a browser login. Smaller fixes too: driver-uploaded docs
now attach to the right load, and driver-recorded stop times cascade up to the
load. Tags v1.15 through v1.20.
